# Manager Checklist

Before moving the task forward, make sure the description answers:

- What problem are we solving?
- Who is affected?
- What should change for the user or system?
- What should stay unchanged?
- Which scenarios must be accepted?
- Which cases are explicitly out of scope?

Recommended structure:

```markdown
## Context
What happened, who is affected, and why this matters.

## Expected result
What should be true after the task is completed.

## Acceptance criteria
- [ ] Concrete, testable criterion.
- [ ] Concrete, testable criterion.
- [ ] Concrete, testable criterion.
```

For bugs, add:

```markdown
## Steps to reproduce
1. ...
2. ...

## Actual result
What happens now.
```

For stories, add:

```markdown
## User story
As a ..., I want ..., so that ...
```
