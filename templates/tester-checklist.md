### Базовый чек-лист для тестировщика

> Автоматическая генерация через OpenAI API недоступна, поэтому использован универсальный шаблон.

## Что проверить

- [ ] Сверить реализацию с разделом `## Ожидаемый результат`.
- [ ] Проверить все пункты из `## Критерии приёмки`.
- [ ] Проверить основной happy path.
- [ ] Проверить негативные сценарии.
- [ ] Проверить граничные значения.
- [ ] Проверить поведение при пустых/некорректных данных.
- [ ] Проверить, что старое поведение не сломалось.
- [ ] Проверить тексты, состояния ошибок и пустые состояния.
- [ ] Проверить права доступа, если задача затрагивает роли пользователей.
- [ ] Проверить API-контракты, если задача затрагивает backend.
- [ ] Проверить логи, метрики или аналитику, если это применимо.

## Шаблон проверки

| № | Шаг | Ожидаемый результат | Статус |
|---|-----|---------------------|--------|
| 1 | Открыть/выполнить основной сценарий | Система работает согласно ожидаемому результату | ⬜ |
| 2 | Повторить сценарий с некорректными данными | Система показывает понятную ошибку | ⬜ |
| 3 | Проверить граничный случай | Система не падает и обрабатывает случай корректно | ⬜ |
| 4 | Проверить регрессию связанного функционала | Старое поведение не сломано | ⬜ |

## Что уточнить перед тестированием

- [ ] Какие сценарии считаются критичными?
- [ ] Какие платформы/браузеры/окружения проверять?
- [ ] Есть ли данные для проверки?
- [ ] Есть ли фича-флаг?
- [ ] Нужно ли проверять аналитику, логи или мониторинги?
