# Portfolio Case: Clarity Guardian

## Краткое описание

Clarity Guardian - это инструмент для повышения качества технической коммуникации в IT-командах.

Он анализирует задачи в GitHub Issues, Pull Requests и Jira, проверяет полноту описания и помогает снизить риск “испорченного телефона” между менеджером, разработчиком и тестировщиком.

Проект создан как практический пример того, как принципы научной коммуникации можно применять в инженерных процессах: делать информацию ясной, структурированной, проверяемой и воспроизводимой.

---

## Проблема

В разработке задача часто становится главным способом передачи знания между ролями.

Если задача описана неясно, каждый участник процесса интерпретирует её по-своему:

- менеджер ожидает один результат;
- разработчик реализует своё понимание;
- тестировщик проверяет неявные предположения;
- итоговый результат расходится с изначальным смыслом.

Это приводит к переделкам, конфликтам ожиданий и потере времени.

---

## Решение

Clarity Guardian автоматически анализирует описание задачи и проверяет:

- наличие контекста;
- ожидаемый результат;
- критерии приёмки;
- стоп-фразы и неоднозначные формулировки;
- соответствие типу задачи: bug, task, story;
- язык шаблона: русский или английский;
- режим проверки: strict или non-strict.

Если описание неполное, бот оставляет комментарий с объяснением проблемы и шаблоном для исправления.

## Clarity Score

В проект добавлена оценка ясности задачи - Clarity Score.

Clarity Score показывает, насколько описание задачи готово к передаче между ролями.

Оценка учитывает:

- структурную полноту описания;
- наличие блокирующих ошибок;
- количество предупреждений;
- коммуникационные риски;
- проверяемость критериев приёмки;
- наличие пользовательского сценария.

Risk Level: medium
